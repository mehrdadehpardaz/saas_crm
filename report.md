گزارش پروژه سیستم پیگیری مشتری (CRM)
خلاصه اجرایی
این پروژه یک سیستم مدیریت ارتباط با مشتری (CRM) تحت وب است که به‌طور خاص برای شرکت‌های کوچک حوزه SaaS طراحی شده است. سیستم با معماری PHP و MySQL توسعه یافته و با رویکرد Mobile-First طراحی شده است تا کاربران بتوانند به‌راحتی از طریق گوشی همراه از آن استفاده کنند.

وضعیت فعلی پروژه
✅ تکمیل شده:
۱. معماری پایگاه داده (Database Schema)
طراحی ۶ جدول اصلی شامل:

users: مدیریت کاربران با ساختار سلسله‌مراتبی (مدیر/کارشناس)

industries: حوزه‌های فعالیت مشتریان با قابلیت توسعه

customers: اطلاعات مشتریان با ارتباط به صنعت

contacts: مخاطبین هر مشتری با قابلیت تعیین مخاطب اصلی

activities: ثبت وقایع و تماس‌ها با تاریخ پیگیری بعدی

subscriptions: مدیریت اشتراک‌ها و پرداخت‌ها

۲. سیستم احراز هویت (Auth)
ثبت‌نام و ورود با شماره موبایل

رمزگذاری امن با password_hash()

اعتبارسنجی کامل ورودی‌ها

مدیریت نشست (Session)

قابلیت تعیین نقش کاربر (ادمین، مدیر، کارشناس)

۳. ساختار فایل‌بندی ماژولار
تفکیک کامل لایه‌ها (MVC ساده):

config/: تنظیمات پایگاه داده

controllers/: منطق برنامه

models/: تعامل با دیتابیس

views/: نمایش به کاربر

assets/: فایل‌های استاتیک (CSS, JS)

includes/: توابع کمکی و هدر/فوتر

۴. طراحی رابط کاربری (UI/UX)
پیاده‌سازی Mobile-First با media queries

استفاده از فونت وزیرمتن برای خوانایی بهتر

کامپوننت‌های مجزا:

متغیرهای CSS برای یکپارچگی رنگ‌ها

فرم‌های بهینه و واکنش‌گرا

کارت‌های اطلاعاتی با سایه‌های نرم

۵. قابلیت Autocomplete برای حوزه فعالیت
کاربر می‌تواند نوع صنعت را تایپ کند

پیشنهادهای مشابه از دیتابیس نمایش داده می‌شود

امکان افزودن صنعت جدید در لحظه

جلوگیری از ثبت موارد تکراری

۶. مدیریت اشتراک (Subscription)
۱۴ روز رایگان برای کاربران جدید

بررسی خودکار انقضای اشتراک

نمایش پیام مناسب در صورت اتمام اشتراک

کاربر می‌تواند اطلاعات را ببیند اما داده‌ها حذف نمی‌شوند
